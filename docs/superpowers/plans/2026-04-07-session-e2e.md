> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 4 つの学習セッション (SRS, Elaboration, Pomodoro, Interleaving) の E2E テストを追加する

**Architecture:** 各テストファイルは `beforeAll` で adminClient 経由のテストデータを作成し、UI フローを実行して `afterAll` でクリーンアップする。既存の `tests/shared/helpers.ts` のファクトリ関数を使用。Pomodoro は `page.clock` でタイマーを高速化する。`playwright.config.ts` で `workers: 1` のため全テストは直列実行され、`cleanupTestData(userId)` による並列テスト間のデータ競合は発生しない。

**Tech Stack:** Playwright, Supabase adminClient, page.clock (Pomodoro)

## UI フロー概要

各テストが辿るフローを図示する。実装時の参照用。

### SRS / Interleaving

```
教材詳細 or Today → StartSessionButton or InterleavingButton
→ /session/[id] (SessionPlayer)
  カード表面 → "めくる" → カード裏面 → 評価 (忘れた/曖昧/正解/簡単)
  → 繰り返し → 完了 → sessionStorage に reviews 保存
→ /session/[id]/review (SessionReview)
  正解数表示 → 自己評価 (SELF_RATING_LABELS) → completeSession
→ /session/[id]/summary
  "セッション完了", カード数, 正答率, 学習時間, 評価分布
```

### Elaboration

```
教材詳細 → StartSessionButton
→ /session/[id] (ElaborationPlayer)
  カード表面 → テキスト入力 → "回答を確認" → カード裏面 + 評価 (SELF_RATING_LABELS)
  → 繰り返し → 完了 → sessionStorage に reviews + elaborations 保存
→ /session/[id]/review (SessionReview)
  回答数表示 → 自己評価 → completeElaborationSession
→ /session/[id]/summary
```

### Pomodoro

```
教材詳細 → StartSessionButton
→ /session/[id] (PomodoroPlayer)
  phase: focus (25分タイマー, setInterval 1000ms)
  → focus_complete: "集中完了" → "5分休憩を開始"
  → break (5分タイマー)
  → break_complete: "休憩完了" → "終了する"
  → done: "学習の振り返り" → 自己評価 → completePomodoroSession
→ /session/[id]/summary
  サイクル数, 集中時間, 学習時間
```

Note: page.clock.install() を使い、fastForward でタイマーを高速化する。

## Task 4: Pomodoro セッション E2E テスト

**Files:**
- Create: `tests/large/session-pomodoro.spec.ts`

**テストデータ:**
- subject (1), material (1), pomodoro method link (カード不要)

**テストフロー:**
1. 教材詳細ページ → StartSessionButton クリック
2. PomodoroPlayer: page.clock.install → fastForward (25分) → "集中完了" → "5分休憩を開始" → fastForward (5分) → "休憩完了" → "終了する" → 自己評価
3. Summary page: サイクル数、集中時間確認

**page.clock の注意点:**
- `page.clock.install()` はページ読み込み前に呼ぶ。setInterval を差し替えるため。
- `page.clock.fastForward(ms)` は期限が来たタイマーコールバックを発火する。
- useCountdownTimer は `setInterval(1000)` を使用。25分 = 1500 コールバック。Playwright が同期的に処理するため性能問題はない。

- [ ] **Step 1: session-pomodoro.spec.ts を作成**

```typescript
// tests/large/session-pomodoro.spec.ts
import { test, expect } from "@playwright/test";
import {
  createTestSubject,
  createTestMaterial,
  getMethodIdBySlug,
  linkMaterialMethod,
  cleanupTestData,
} from "./helpers/db";
import { getTestUser } from "./helpers/types";

// src/lib/constants.ts と同じ値
const POMODORO_FOCUS_SEC = 1500;
const POMODORO_BREAK_SEC = 300;

test.describe("Pomodoro セッション", () => {
  let userId: string;
  let materialId: string;

  test.beforeAll(async () => {
    const user = getTestUser();
    userId = user.id;
    const subject = await createTestSubject(userId, `E2E-Pomo-${Date.now()}`);
    const material = await createTestMaterial(subject.id, userId, "Pomodoro テスト教材");
    materialId = material.id;
    const pomodoroMethodId = await getMethodIdBySlug("pomodoro");
    await linkMaterialMethod(material.id, pomodoroMethodId);
  });

  test.afterAll(async () => {
    await cleanupTestData(userId);
  });

  test("タイマー → 休憩 → 終了 → 自己評価 → サマリー", async ({ page }) => {
    // page.clock は setInterval を差し替えるため、ページ読み込み前に install する
    await page.clock.install();

    await page.goto(`/materials/${materialId}`);

    // セッション開始
    // seed data: learning_methods.name = "ポモドーロ"
    await page.getByRole("button", { name: /ポモドーロ/ }).click();
    await page.waitForURL(/\/session\/[\w-]+$/, { timeout: 10_000 });

    // Focus phase: タイマーが表示される
    await expect(page.getByText("集中タイマー")).toBeVisible();

    // Focus 完了まで時間を進める
    // fastForward 後、React の状態更新を待つため toBeVisible の timeout を設ける
    await page.clock.fastForward(POMODORO_FOCUS_SEC * 1000);
    await expect(page.getByText("集中完了")).toBeVisible({ timeout: 5_000 });

    // 休憩開始
    await page.getByRole("button", { name: /休憩を開始/ }).click();
    await expect(page.getByText("休憩タイマー")).toBeVisible();

    // Break 完了まで時間を進める
    await page.clock.fastForward(POMODORO_BREAK_SEC * 1000);
    await expect(page.getByText("休憩完了")).toBeVisible({ timeout: 5_000 });

    // 終了する
    await page.getByRole("button", { name: "終了する" }).click();

    // Self-rating (inline in PomodoroPlayer, phase="done")
    // Pomodoro のボタンテキストは "{r}. {SELF_RATING_LABELS[r]}" 形式 (例: "3. おおむね理解できた")
    await expect(page.getByText("学習の振り返り")).toBeVisible();
    await page.getByRole("button", { name: /おおむね理解できた/ }).click();

    // Summary page (status = 'completed' でないと notFound)
    await page.waitForURL(/\/session\/[\w-]+\/summary/, { timeout: 10_000 });
    await expect(page.getByText("セッション完了")).toBeVisible();
    await expect(page.getByText("サイクル数")).toBeVisible();
    await expect(page.getByText("集中時間")).toBeVisible();
  });
});
```

## Troubleshooting

### StartSessionButton のセレクタが見つからない

教材詳細ページの StartSessionButton の label は DB の `learning_methods.name` に依存する。テスト実行前に seed data のメソッド名を確認:

```sql
SELECT slug, name FROM learning_methods;
```

結果に応じて `getByRole("button", { name: /.../ })` の正規表現を調整する。

### Pomodoro の page.clock.fastForward が遅い / タイムアウト

1500 回の setInterval コールバックが同期実行されるため、環境によっては時間がかかる。代替アプローチ:

```typescript
// より小刻みに fastForward (React の再レンダリングを挟む)
for (let i = 0; i < 15; i++) {
  await page.clock.fastForward(100_000); // 100秒ずつ
}
```

### Interleaving ボタンが Today ページに表示されない

`InterleavingButton` は `materials.length >= 2` のときのみ表示。2 教材とも `getDueMaterials()` が返す条件を満たす必要がある:
- SRS method がリンクされている
- `srs_states.due_date` が現在時刻以前のカードが存在する

### Review ページで "読み込み中..." のまま

SessionPlayer / ElaborationPlayer は `sessionStorage` にレビューデータを保存して review ページに遷移する。review ページは `sessionStorage` からデータを読む。`page.clock.install()` 後にこの遷移が失敗する場合、clock が `sessionStorage` の動作に影響している可能性がある。その場合、clock.install は session ページ遷移後に実行する。
